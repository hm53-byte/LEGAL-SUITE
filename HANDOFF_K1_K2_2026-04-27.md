## Sljedeći prioriteti (po dogovoru s korisnikom)

### 0. K1.5 — UI ožičenje (PRIORITET 0, sad)

**Problem**: K1 audit chain je opt-in. `pripremi_za_docx()` ima nove parametre `input_dict` i `generator_module_path`, ali `stranice/*.py` ih trenutno **ne prosljeđuju**. Bez ovog koraka K1 je mrtvi kod u repu.

**Akcija**: rutinski refaktor (NE kroz BRZ_MOZAK — jasan task, nema arhitekturne odluke). Pun pregled svih `stranice/*.py` forma:

```python
# Trenutno:
doc_html = generiraj_kupoprodajni_ugovor(podaci)
prikazi_dokument(doc_html, "Kupoprodajni_ugovor.docx", "Preuzmi")

# Treba:
doc_html = generiraj_kupoprodajni_ugovor(podaci)
prikazi_dokument(
    doc_html,
    "Kupoprodajni_ugovor.docx",
    "Preuzmi",
    doc_type="ugovor",
    doc_subtype="kupoprodajni",
    input_dict=podaci,  # K1: dict s user input-ima
    generator_module_path="generatori/ugovori.py",  # K1: za bytecode hash
)
```

**Stranice koje treba ožičiti** (15 + jednostavni mod):
- `stranice/ugovori.py`, `tuzbe.py`, `ovrhe.py`, `zalbe.py`, `zemljisne.py`, `opomene.py`, `punomoci.py`, `trgovacko.py`, `obvezno.py`, `obiteljsko.py`, `upravno.py`, `kazneno.py`, `stecajno.py`, `potrosaci.py`, `posrednik_najam.py`, `jednostavno.py`

Trebat će i prilagoditi `pomocne.prikazi_dokument()` da prima novi `input_dict` parametar i prosljeđuje u `pripremi_za_docx`. Provjeri točno gdje se zove `pripremi_za_docx` — ako je preko `prikazi_dokument`, sve prolazi kroz jedan helper i refaktor je čišći.

**LOC procjena**: ~30 LOC po stranici × 16 stranica = ~480 LOC izmjena (ali tipično 5-10 LOC po pozivu, ne ekspanzija). **Solo dev rok**: ~2-3 dana ako se radi jedan-po-jedan; ~1 dan ako se napravi batch refactor s search/replace pattern-a.

**Test plan**: postojeći 194 testovi moraju ostati zeleni. Plus po jedan smoke test po stranici da generira docx s prosljeđenim `input_dict` i provjeri da se `download_log` red puni s K1 audit poljima (kad je Supabase konfiguriran; kad nije, tihi fall-through ostaje).

**Acceptance**: nakon K1.5, svaki docx download generira download_log red s popunjenim `input_canonical_hash`, `output_sha256`, `parent_hash`, `current_hash`, `generator_version_hash`, `input_schema_version`. Verify hash chain s `verify_chain([rows]) → (True, None)`.

## Što SE PRIPAZILO da se ne razbije

- **Backwards compatibility**: svi novi parametri u `pripremi_za_docx`, `record_download`, `html_u_docx` su opcijski s default `None`. Stari pozivi rade bez izmjene.
- **Graceful degradation**: ako Supabase ne odgovara, `entitlements` funkcije vraćaju free tier; ako `audit_chain` import fail-a, `docx_export` nastavlja s K3 watermarkom; ako `watermark` import fail-a, `docx_export` nastavlja bez forensic ID-a (samo K3 NACRT watermark).
- **Idempotent migrations**: `cloud/0007_audit_chain.sql` koristi `ADD COLUMN IF NOT EXISTS` (Postgres 9.6+); siguran ponovni run.
- **Feature-flag-less rollout**: K1 audit chain je opt-in preko parametara (ne env varijable), tako da nema "globalni flag" koji bi mogao razbiti pola app-a; svaki poziv se posebno opt-ina.
